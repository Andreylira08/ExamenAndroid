## Realiza un escrito que alinee las etapas de desarrollo de software

Para hablar sobre las etapas de desarrollo de software, utilizare las metodolgías que aplique en el desarrollo de este examen. Primero hablemos de las etapas:

1. Analisis
2. Definir
3. Idear
4. Prototipar
5. Probar
6. Desplegar

En la etapa de análisis, mi objetivo fue comprender cómo funcionaba la API del examen. Aunque tenía un objetivo similar al de la Pokedex de los laboratorios, la API del examen se comportaba de manera diferente. Para entender su funcionamiento, utilicé POSTMAN para realizar llamadas GET, analizar los datos disponibles y determinar cómo integrarlos correctamente. Esta fase fue más lenta de lo esperado, pero fue fundamental para lograr un producto sólido.

Durante la etapa de definición, planifiqué la estructura del proyecto. Decidí implementar MVVM combinado con Clean Architecture, tomando como referencia los laboratorios del profesor sobre la Pokedex. Esta decisión permitió organizar mejor el código y separar responsabilidades de manera clara.

En la etapa de ideación y prototipado, integré todo lo definido y analizado, desarrollando la aplicación mientras realizaba pruebas continuas. Aquí fue donde la entro la metodologia aguil: a mitad del proceso tuve que reestructurar la integración con la API, lo que me llevó a volver temporalmente al análisis y luego retomar el prototipado, siempre adaptándome a los cambios y priorizando un producto completo y funcional.

Finalmente, en las etapas de prueba y despliegue, me aseguré de que la aplicación cumpliera con todos los requisitos funcionales del examen. Una vez verificado que se cumplían los objetivos y completados los puntos extra del proyecto, se procedió a crear el Pull Request y preparar la aplicación para su despliegue.

Este enfoque permitió mantener un ciclo iterativo de mejora continua, asegurando que el producto final fuera estable, funcional y alineado con los requisitos planteados desde el inicio.
